From curiosity to practical AI delivery

Over the past four months, my coding journey has moved quickly from curiosity into real hands-on delivery. What started as experiments with VS Code, GitHub Copilot Pro, and GitHub Pages became a practical workflow for building personal webpages, training projects, AI agents, and capstone work.

This page documents that transition: the tools I tried, the lessons I kept, and how AI-assisted coding became part of my daily build process.

February 2026

February 2026: Starting with VS Code and GitHub Copilot Pro

I began by setting up VS Code as my main development environment and using GitHub Copilot Pro to help write, debug, and understand code faster. Around the same time, I started using GitHub to host and publish my webpages, turning ideas into visible projects instead of keeping them as local experiments.

VS Code GitHub Copilot Pro GitHub Pages
March to Mid-May 2026

March to Mid-May 2026: Building with Claude Code Pro

In March, I subscribed to Anthropic Claude Code Pro for a three-month learning cycle. Claude helped me move faster on personal and training projects, especially when planning page structures, improving code, and experimenting with AI-assisted development patterns. This period helped me understand what an AI coding partner could do well: explain, refactor, troubleshoot, and keep projects moving.

Claude Code Pro Training projects AI-assisted refactoring
Late April 2026

Late April 2026: Testing OpenCode with free models

In the last week of April, I started trying OpenCode inside my VS Code vibe-coding workflow. For two to three weeks, I tested different free models, including MinMax, and also played with DeepSeek V4 through OpenCode while building small projects. This gave me a practical way to compare model behavior, response quality, coding speed, and how each assistant handled everyday project work.

OpenCode MinMax DeepSeek V4 Small project tests
Mid-May 2026

Mid-May 2026: Switching toward OpenAI Codex

In mid-May, I was given access to OpenAI Codex Pro 20x and started shifting more of my workflow from Claude to Codex. This also became the point where I began cleaning up Anthropic Claude API and AI agent code, replacing parts of the stack with OpenAI-based implementations. Codex is now helping me vibe code my capstone projects, refine my portfolio, and turn experiments into more complete systems.

OpenAI Codex Pro 20x OpenAI API Capstone projects

The journey continues

Each project teaches me something new about prompts, agents, APIs, automation, debugging, and how to work with AI without losing ownership of the final result. My goal is not just to use AI to write code faster, but to become better at designing, testing, explaining, and shipping useful systems.
